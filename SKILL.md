# 周报/日报自动生成

## 任务目标
- 根据用户提供的工作记录，自动生成结构化的工作报告
- 支持三种报告模式：日报、周报、自定义时间段
- 自动识别用户需要的模式，或让用户选择

## 触发条件
当用户提到以下关键词时触发本技能：
- "生成日报"、"写日报"、"日报"
- "生成周报"、"写周报"、"周报"
- "工作报告"、"工作汇报"、"工作总结"
- "汇报工作"、"写报告"
- 用户提供了工作记录并要求整理成报告格式

## 执行流程

### 第一步：识别模式
根据用户输入判断报告模式：
- 如果用户明确说"日报"或只提供了今天的工作 → **日报模式**
- 如果用户明确说"周报"或提到"本周"/"上周" → **周报模式**
- 如果用户指定了自定义时间段 → **自定义模式**
- 如果不确定，询问用户需要哪种模式

### 第二步：信息收集
分析用户提供的工作记录，提取关键信息。如果信息不足，主动提问引导用户补充。

**需要引导用户补充的信息：**
- 完成了哪些工作？（必填）
- 正在进行的工作有哪些？（可选）
- 遇到什么问题？（可选）
- 明天/下周计划做什么？（可选）

### 第三步：生成报告

#### 日报模式
生成结构：

```
【日报】YYYY-MM-DD

📋 今日完成
• 逐条列出已完成的工作项
• 包含具体的成果和数据

🔄 进行中
• 未完成但正在推进的工作

📝 明日计划
• 计划明天要做的工作

⚠️ 今日问题
• 遇到的问题及解决方案（如果用户提供了）

📊 工作状态评估
• 整体效率评估：高 / 中 / 低
• 简要评语
• 工作饱和度：饱和 / 正常 / 不饱和
```

#### 周报模式
生成结构：

```
【周报】YYYY-MM-DD ~ YYYY-MM-DD

✅ 本周完成
• 逐条列出本周完成的工作
• 标注关键成果

🔄 进行中
• 正在进行的工作

📋 下周计划
• 计划下周开展的工作

🏆 主要成果
• 本周最核心的2-3个成果
• 量化数据优先

📊 总体评价
• 进度状态：正常 / 滞后 / 超前
• 工作表现综合评价
• 改进建议（如有）
```

#### 自定义模式
生成结构：

```
【工作报告】YYYY-MM-DD ~ YYYY-MM-DD

📋 完成事项
• 逐条列出

🔄 进行中工作
• 逐条列出

📝 待办计划
• 逐条列出

⭐ 重点亮点
• 突出成果

💡 总结
• 总体评价与建议
```

### 第四步：格式优化
- 使用简洁的 Markdown 格式
- 同一类内容用项目符号（•）列出
- 日期使用标准格式 YYYY-MM-DD
- 数据和成果用**加粗**突出
- 保持可读性，避免过长段落

## 特殊场景处理

### 用户只提供了零散信息
- 主动分类整理：帮用户识别哪些是"已完成"、哪些是"进行中"、哪些是"待办"
- 输出时标注"根据您提供的信息整理，如有遗漏请补充"

### 用户提供的信息很少
- 先用已有信息生成框架
- 在每个部分标注"（待补充）"引导用户完善
- 例如："根据您提供的信息生成初稿，您可以在以下部分补充更多细节：..."

### 用户提供了大量信息
- 按类别智能归类
- 合并相似的工作项
- 提取最重要的3-5个关键成果
- 在报告末尾可以加"💡 数据洞察"部分做简要分析

### 中英文混合场景
- 如果用户输入包含英文，报告主体用中文，但保留英文术语原名
- 如果用户全程使用英文，输出英文报告
- 输出的日期格式保持统一

## 输出示例

### 日报示例输出

```
【日报】2026-05-11

📋 今日完成
• 完成了用户管理模块的API开发，新增5个接口
• 修复了3个生产环境bug（订单超时、登录闪退、数据导出乱码）
• 完成团队周会纪要整理并同步全员

🔄 进行中
• 订单系统性能优化方案设计（预计本周完成）

📝 明日计划
• 推进订单系统优化方案评审
• 开始数据库索引优化

⚠️ 今日问题
• 测试环境数据库连接不稳定，已联系DBA处理

📊 工作状态评估
• 整体效率：高
• 饱和度：饱和
• 评语：今日高效完成了多项任务，建议明日聚焦订单系统优化
```

### 周报示例输出

```
【周报】2026-05-05 ~ 2026-05-11

✅ 本周完成
• **用户管理模块**：完成API开发，包含5个RESTful接口
• **Bug修复**：修复3个生产环境bug（订单超时、登录闪退、数据导出乱码）
• **技术文档**：完成API接口文档v2.0更新

🔄 进行中
• 订单系统性能优化方案设计（进度70%）

📋 下周计划
• 订单系统优化方案评审与实施
• 数据库索引优化
• 开始准备Q2技术复盘材料

🏆 主要成果
1. 用户管理模块按时交付，进度提前2天
2. 生产环境bug全部修复，系统稳定性提升

📊 总体评价
• 进度状态：正常
• 综合评价：本周工作按计划推进，用户管理模块交付质量良好。建议下周聚焦性能优化。
```
